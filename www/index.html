Static files

Uploads

Multipart and raw uploads, listing, and delete — all in one page.

Read-only directory

location /readonly — GET only, autoindex on

Same autoindex mechanism, but the route only allows GET. Good for a quick 405 check.

  • /readonly/ — directory listing

Redirect

location /redirect — GET, return 302 /

A route that never serves a file, only a Location header.

  • /redirect — follow it, ends up back here

Error pages cheat sheet

Each status below has a custom page configured via error_page in config/default.conf. Some are one click away, others need a raw request since browsers won't send a malformed one for you.

CodeWhen it firesTry it
400 Malformed request line (e.g. a lowercase HTTP method). curl -X get http://localhost:8080/
403 File exists but can't be read, or a directory has autoindex off with no index file. chmod 000 a file under www/, then request it
404 No route/file matches the request path. /this-page-does-not-exist
405 Route matched, but the method isn't in its allow_methods.
408 Client opens a connection and stays idle for more than 30s. nc localhost 8080 then wait ~30s
413 Body larger than client_max_body_size (10 MiB here).
500 Server-side failure (e.g. upload directory missing/unwritable). configuration-dependent, no safe live trigger
501 Defined for completeness; the request parser currently rejects any method outside GET/POST/DELETE with 405 before this path is reached. view page
